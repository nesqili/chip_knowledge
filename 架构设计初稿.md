# 芯片成本学习页面 - 架构设计（v2，金字塔结构）

## 1. 目标与用户
- 主要用户：刚入门的产品经理，零基础学习芯片经济学。
- 对标芯片（边缘侧 SoC）：以 Rockchip **RK3566 / RK3588** 一类的边缘侧 SoC 作为成本模型的默认对标对象（也可切换到 **RK3568 / RK3576 / Allwinner T527 / Amlogic A311D2** 等同级别竞品）。
  - RK3566：4×A55，22nm，中低端边缘 AIoT SoC，量产单颗成本量级约 5–8 美元。
  - RK3588：4×A76 + 4×A55 + 6 TOPS NPU，8nm（Samsung），高端边缘 SoC，量产单颗成本量级约 25–40 美元。
  - 以上数值作为计算器默认值与学习示例的锚点，而非承诺价格。
- 一张页面支持三种使用方式：
  1. 学习 - 按"金字塔"从战略层 → 结构层 → 明细层逐级下钻。
  2. 查阅 - 可搜索、可筛选的成本项与供应商目录。
  3. 估算 - 交互式计算器，基于行业通用公式输出大致单颗成本，支持在 RK3566 / RK3588 两套默认参数之间一键切换。
- 交付形式：位于 `/Users/kerry/Qoder/Chip/` 目录下的单一离线 HTML 文件 `index.html`，内嵌 CSS/JS/JSON 数据，双击即可打开，无需构建工具，无需后端。

## 1.5 金字塔结构（核心叙事骨架）

整个页面、学习路径、数据模型都围绕一张四层金字塔组织：

```
                 ┌──────────────────────────┐
         L1      │   战略层 / 产品级总成本   │   ← "一颗 RK3588 值多少钱"
                 │   Unit Cost / BOM / ASP   │
                 └─────────────┬────────────┘
                               │
              ┌────────────────┼────────────────┐
      L2      │   结构层 / 五大成本域             │   ← "钱花在哪几块"
              │  Silicon Die | Package | Test |  │
              │  NRE 摊销     | IP 版税 | 综合   │
              └────────────────┬────────────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
   L3   │   明细层 / 约 20 个成本项                      │   ← "每块里又分哪些"
        │  晶圆、掩膜、DPW、良率、基板、CP、FT、版税 …    │
        └──────────────────────┬──────────────────────┘
                               │
                 ┌─────────────┴────────────┐
         L4      │   供给层 / 供应商 + 公式   │   ← "谁供的、怎么算出来的"
                 │  TSMC/三星/ASE/Arm/…       │
                 └──────────────────────────┘
```

四层之间的关系：
- L1 告诉 PM「一颗芯片要卖多少钱 / 成本多少」，用于对齐商业直觉。
- L2 把 L1 拆成**五大成本域**，每个域在页面上是一个可点开的卡片。
- L3 把每个域拆成具体**成本项**（Cost Item），是「查阅」的主要对象。
- L4 给每个成本项挂上**供应商**与**计算公式**，是「估算」的数据源。

页面的三种使用方式分别命中不同层：
- 学习：从 L1 向下钻到 L4（金字塔导览）。
- 查阅：在 L3 / L4 横向搜索筛选。
- 估算：用 L4 的公式反推 L1 的结果。

## 2. 内容模型（页面要教的东西）

### 2.1 L2 五大成本域（一级拆解）
在金字塔的 L2 层，我们把任何一颗边缘 SoC 的出厂成本固定拆成五个域，用于教学与占比可视化：

| 代号 | 成本域 | 典型占比（边缘 SoC） | 关键驱动因素 |
| --- | --- | --- | --- |
| D1 | Silicon Die（硅片成本） | 45%–60% | 工艺节点、Die 面积、良率 |
| D2 | Package（封装） | 10%–20% | 封装形式（FCBGA/LFBGA/QFN）、基板层数、Pin 数 |
| D3 | Test（测试） | 5%–10% | CP/FT 测试时间、ATE 机时单价 |
| D4 | NRE 摊销 | 5%–15% | 掩膜版费用、设计投入、总摊销销量 |
| D5 | IP 版税 + 综合费用 | 10%–20% | Arm 版税率、物流、RMA、公司毛利 |

### 2.2 L3 成本明细项（约 20 个，挂到 5 个域下）
- D1 Silicon Die
  - 晶圆价格（按工艺节点：22nm / 12nm / 8nm / 6nm …）
  - Die 面积（RK3566 ≈ 55–65 mm²，RK3588 ≈ 105–115 mm²，示意值）
  - DPW（每片晶圆出 Die 数）
  - 缺陷密度 D0 与良率
  - 晶圆级测试 CP
- D2 Package
  - 基板 / 载板（FCBGA 层数、BT / ABF 材料）
  - 引线框 / 硅中介层（高端 SoC 一般不用中介层，HPC 才用）
  - OSAT 封装工序（倒装、植球）
- D3 Test
  - 终测 FT（ATE 机时 × 时间）
  - 系统级测试 SLT（RK3588 类高端 SoC 更常见）
  - 老化 Burn-in（车规、工规时才上）
- D4 NRE 摊销
  - 掩膜版 / 光罩 Mask Set（22nm ~1–2M USD，8nm ~10–15M USD，示意）
  - EDA 工具 License
  - 设计团队工资 + 设计服务外包
  - 原型片、点亮、特性化
  - 质量认证（JEDEC / AEC-Q / ISO 26262 / 3C / FCC）
- D5 IP 版税 + 综合
  - Arm CPU 版税（按颗计费）
  - GPU / NPU / ISP / 接口 IP 版税
  - 物流、仓储、关税
  - 失效 / RMA 准备金、报废
  - 公司毛利、代理商毛利

### 2.3 L4 供应商地图（按环节）
- IP 厂商：Arm、Synopsys、Cadence、Imagination、SiFive、Ceva、Alphawave
- EDA 厂商：Synopsys、Cadence、Siemens EDA（原 Mentor）
- 晶圆代工：TSMC、三星代工、Intel Foundry、GlobalFoundries、联电（UMC）、中芯国际（SMIC）
- 光罩：Photronics、DNP、Toppan
- 硅片：信越化学（Shin-Etsu）、SUMCO、Siltronic、环球晶圆（GlobalWafers）、SK Siltron
- 设备（上下文了解即可）：ASML、应用材料（AMAT）、Lam Research、KLA、东京电子（TEL）
- OSAT 封测：日月光（ASE）、安靠（Amkor）、长电科技（JCET）、矽品（SPIL）、力成（PTI）、通富微电
- 封装基板：欣兴（Unimicron）、揖斐电（Ibiden）、新光电气（Shinko）、景硕（Kinsus）、AT&S、三星电机（Semco）
- 测试设备（ATE）：泰瑞达（Teradyne）、爱德万（Advantest）、Cohu
- 特种化学品/气体：液化空气、林德、默克 / EMD、Entegris

每个供应商条目包含：名称、所在环节、地区、典型成本杠杆、备注。

### 2.4 L4 成本计算（既作为学习内容展示，也驱动计算器）
- 每片晶圆出 Die 数（DPW）：`DPW = π·(D/2)² / S − π·D / √(2·S)`，其中 D 为晶圆直径（mm），S 为 Die 面积（mm²）。
- 良率（Murphy 模型）：`Y = ((1 − exp(−A·D0)) / (A·D0))²`，其中 A 为 Die 面积，D0 为缺陷密度（/cm²）。
- 每片晶圆出好 Die 数：`GDW = DPW × Y`。
- 单颗 Die 成本：`Die$ = 晶圆单价 / GDW`。
- 单颗出厂成本：`Unit$ = Die$ + 封装$ + 测试$ + IP 版税$ + NRE$ / 总销量 + 综合费用%`。
- ASP（平均售价）与单颗成本之差 → 毛利率。

### 2.5 对标芯片默认参数（计算器预置）
| 参数 | RK3566（22nm） | RK3588（8nm） |
| --- | --- | --- |
| 工艺节点 | 22nm ULL（SMIC/UMC 量级） | 8nm LPP（Samsung 量级） |
| Die 面积 | ~60 mm² | ~110 mm² |
| 晶圆单价（12 英寸） | ~USD 2,500 | ~USD 6,500 |
| 缺陷密度 D0 | 0.15 /cm² | 0.10 /cm² |
| 掩膜版一次性 | ~USD 1.5M | ~USD 12M |
| 封装 | LFBGA / FCBGA | FCBGA（大尺寸） |
| 封装单价 | ~USD 0.6 | ~USD 2.5 |
| 测试单价 | ~USD 0.3 | ~USD 1.2 |
| IP 版税 | ~USD 0.3 | ~USD 1.5 |
| 预期年销量 | 8M | 3M |

以上均为**公开资料级别的量级估算**，页面中会标注「仅作教学示意」。

## 3. 页面架构

### 3.1 总体布局（单页，多 Tab，按金字塔组织）
- 顶栏：标题、全局搜索框（对所有内容做模糊搜索）、对标芯片切换（RK3566 / RK3588 / 自定义）、明/暗主题切换。
- 左侧导航（移动端可折叠），采用金字塔逐层导览：
  - L1 概览 / L2 五大成本域 / L3 成本项 / L4 供应商 / 计算器 / 公式推导 / 术语表。
- 主内容区按当前 Tab 渲染，默认进入 L1 概览。
- 页脚：免责声明（数值仅作教学示意、非官方报价）。

### 3.2 各 Tab 内容（与金字塔层级对齐）
1. **L1 概览**
   - 金字塔全景 SVG（可点击 L2→L3→L4 级联高亮）。
   - 对标芯片简介卡：RK3566 / RK3588 规格、典型应用场景、BOM 位置。
   - “一张图看完成本”占比饼图（内嵌 SVG），鼠标悬停显示每块叠加明细。
2. **L2 五大成本域**
   - 五张横向卡片（D1–D5），每张显示占比条、关键驱动因素、代表性供应商 Logo。
   - 点击卡片 → 平滑滚动到 L3 并预置该域的筛选。
3. **L3 成本明细项**
   - 可筛选表格（筛选器：所属域 D1–D5、所处阶段、NRE/经常性）。
   - 点击某行→侧滑抽屉：定义、驱动因素、典型区间、相关供应商、RK3566 / RK3588 示意数值。
4. **L4 供应商**
   - 可筛选表格（环节、地区），每个供应商展示其驱动的成本项（反向指向 L3）。
5. **计算器**
   - 顶部对标选择器：`RK3566 默认` / `RK3588 默认` / `自定义`。
   - 表单输入 → 实时输出 DPW、良率、Die 成本、单颗成本、毛利率；同时展示占比饼图（与 L2 一致的五块）。
   - 提供敏感度小条（Die 面积 ±10%、D0 ±20%、销量 ±30%）。
6. **公式推导**：公式来源与 RK3588 的一份完整示例计算（从晶圆价到单颗出厂）。
7. **术语表**：按字母排序（Tape-out、Reticle、KGD、Yield ramp、Wafer bank 等）。

### 3.2.1 结构化拆解图（重点视觉元素，均为内嵌 SVG）
- **金字塔全景图**（L1 默认展示）：四层梯形，点击任一层 → 跳转对应 Tab。
- **成本叠片条（Stacked Bar）**：RK3566 vs RK3588 并排，清晰展示不同工艺 / 规格下五大成本域的给定占比差异。
- **价值链流程图**：设计 → IP/EDA → 光罩 → 代工 → 封测 → 分销，每一环节下挂典型供应商。
- **思维导图Mindmap**：在 L3 Tab 顶部，用一张三级树状图展示 D1–D5 与其下 20 个成本项的关系（纯 SVG，无依赖）。

### 3.3 数据层（严格映射金字塔层级）
- 所有内容以 JS 常量放在文件顶部：
  - `DOMAINS = [{id:'D1'…'D5', name, pct, drivers, description}]`                         // L2
  - `COST_ITEMS = [{id, name, domainId:'D1'…'D5', phase, nreOrRecurring, drivers, typicalRange, description, supplierIds:[]}]`  // L3
  - `SUPPLIERS = [{id, name, layer, region, role, notes, relatedItemIds:[]}]`    // L4
  - `TARGET_CHIPS = [{id:'RK3566'|'RK3588', name, node, dieAreaMm2, waferPriceUsd, d0, maskSetUsd, pkgUsd, testUsd, royaltyUsd, annualVolume}]` // 对标芯片默认值
  - `GLOSSARY = [{term, definition}]`
- 页面加载时基于以上数组构建搜索索引；全局搜索返回按类型分组的结果（域 / 成本项 / 供应商 / 术语）。

### 3.4 交互与样式
- 纯原生 JavaScript（不引入任何框架），保证文件可移植。
- 样式：单份内嵌 CSS，系统字体栈，通过 `prefers-color-scheme` + 手动开关支持明暗主题，移动端优先。
- 不依赖任何外部 CDN（完全离线可用）。图标使用内嵌 SVG。
